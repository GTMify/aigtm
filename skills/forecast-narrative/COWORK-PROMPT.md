# Forecast Narrative — Cowork Prompt

```
Turn my pipeline data into a forecast narrative I can send to my [manager / CRO / board].

## My numbers
Quota this [month / quarter]: $[amount]
Days remaining: [N]

## What I need
1. A commit number backed by named deals with specific confidence signals (not vibes)
2. Upside deals with what needs to happen for each to close
3. Risks flagged with what I'm doing about each one
4. Deals I lost or pushed since last update, with reasons
5. A gap analysis: if commit doesn't cover quota, how much net-new do I need?
6. An "ask" section if I need anything from leadership
7. A one-sentence honest bottom line

## Tone
[Confident but honest / Conservative / Aggressive — pick one]
This is going to: [my manager in our weekly 1:1 / the CRO in forecast call / a board deck]

## Rules
- Don't inflate confidence. If my pipeline doesn't support my number, tell me.
- Show the math. Every commit should trace to a named deal.
- Distinguish facts ("verbal yes from the VP") from hopes ("they seemed excited").
- Keep the whole thing under one page.

## Pipeline Data
[Paste your deals — CRM export, list of deals with stage/amount/close date, or free text]
```
